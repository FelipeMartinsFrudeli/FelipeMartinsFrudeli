### What I do

I enter messy technical environments, define clearer boundaries, and move the system forward with less risk. The work usually looks like one of these:

- **Stabilize fragile backend services** in TypeScript / Node.js (NestJS, Express), with DDD-driven boundaries and integration patterns that reduce brittle coupling.
- **Ship fullstack product** end-to-end across Vue / React Native / Kotlin when the business needs delivery, not just architecture diagrams.
- **Stand up agent infrastructure** — multi-agent orchestration, governance, and tooling on top of the open-source [Paperclip](https://github.com/paperclipai/paperclip) platform.
- **Build hardware-adjacent software** for FRD Technologies: phone-as-hub cognitive performance tools, sensor fusion, on-device transcription, lightweight HUD integrations.

The aim is the same in every contract: changes that are safe enough for the business to keep moving.

### How I think about architecture

Patterns are tools for buyer outcomes, not signals of taste. I reach for DDD when business boundaries are blurry and integrations keep breaking; for hexagonal when a system is locking itself into one framework; for enterprise integration patterns when contracts between services are doing damage. The goal is always the same: less risk to change, less operational rework, clearer decisions for the people running the business.
